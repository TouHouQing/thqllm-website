# 账户、额度与使用记录

账户相关数据会随使用情况和控制台规则变化。余额、额度、价格、账单、限制和可用操作均以 [THQ API 控制台](https://sub.thqllm.com) 当前显示为准。

## 账户页面能做什么

账户页面用于确认当前账户状态，并进入与额度、使用记录、API Key 和账单相关的操作。页面名称和可用功能可能调整，请按控制台实际显示的入口操作。

开始接入前，至少确认以下信息：

| 项目 | 需要确认的内容 |
| --- | --- |
| 账户 | 当前登录的是预期账户 |
| 额度 | 可用额度足以完成验证请求 |
| 模型 | 目标模型当前可用，模型 ID 与协议匹配 |
| API Key | Key 属于当前用途，未被撤销或停用 |
| 限制 | 当前模型和账户的速率、并发或其他限制 |

## 额度与扣费

额度不是固定的请求次数。一次请求的实际消耗可能受模型、输入、输出和控制台当前计费规则影响。

- 发起批量任务前，先核对目标模型当前价格和账户可用额度。
- 比较成本时使用同一模型、相近输入和相同任务条件。
- 发现消耗异常时，先暂停相关客户端或服务，避免在排查期间继续产生请求。
- 不要用本文中的示例推算价格；动态价格和计费口径以控制台为准。

## 限制与撤销 API Key

当某个 Key 不再使用、用途不明或可能泄露时，应在控制台撤销，并为仍需运行的服务创建新 Key。

撤销前先确认哪些客户端依赖该 Key。轮换时逐个更新并执行最小请求，避免无法判断是哪一处配置仍在使用旧 Key。

不要用一个 Key 连接所有设备和项目。用途隔离可以让撤销、成本归因和异常排查更直接。

## 充值或账单异常

充值状态、账单明细、支付方式和处理入口以控制台当前显示为准。遇到异常时：

1. 不要连续重复提交同一笔充值。
2. 保存订单时间、订单标识、支付状态和控制台显示结果。
3. 核对使用记录中是否存在非预期请求。
4. 通过控制台当前可见的账单或支持入口提交问题。

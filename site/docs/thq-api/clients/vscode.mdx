# VS Code 接入 THQ API

VS Code 是扩展平台，不是统一的模型客户端。不同 AI 扩展可能使用 OpenAI Chat Completions、
OpenAI Responses、Anthropic Messages 或其他协议，也可能完全不允许修改服务地址。开始配置前，
必须先确认具体扩展公开支持的协议和自定义端点能力。

模型 ID 和可用能力会动态调整，请从
[THQ API 控制台](https://sub.thqllm.com)复制当前可用的完整模型 ID。

## 先确认扩展协议

在扩展的官方文档和设置中确认以下项目：

| 检查项 | 需要确认的内容 |
| --- | --- |
| 协议 | 使用 Chat Completions、Responses 还是 Anthropic Messages |
| 自定义地址 | 是否提供 Base URL、API Host 或 Provider 配置 |
| 认证 | 是否支持 API Key 或环境变量 |
| 模型 | 是否允许填写自定义模型 ID |
| 配置范围 | 配置保存在 VS Code 设置、用户级文件还是项目文件 |

不要把“支持 OpenAI 模型”理解成“支持任意 OpenAI 兼容地址”。如果扩展没有公开的 Base URL
或自定义 provider 设置，就不要通过不受支持的字段强行接入。

### 官方 Codex 扩展

官方 Codex IDE 扩展与 Codex CLI 共享用户级 `~/.codex/config.toml`。当前自定义 provider
使用 OpenAI Responses API，因此 THQ Base URL 应为 `https://api.thqllm.com/v1`，并且所选
模型和服务端必须支持最终的 `/v1/responses` 请求。

这不等同于任意 VS Code 扩展都能使用 Responses。完整配置见
[Codex 接入指南](/docs/thq-api/clients/codex)，扩展行为见
[OpenAI Codex IDE 官方文档](https://developers.openai.com/codex/ide/)和
[Codex 配置官方文档](https://developers.openai.com/codex/config-basic/)。

## OpenAI 兼容扩展

仅当扩展明确允许添加 OpenAI 兼容 provider，并能配置 Base URL、API Key 和模型时，使用：

| 字段 | 值 |
| --- | --- |
| Base URL | `https://api.thqllm.com/v1` |
| API Key | `YOUR_THQ_API_KEY`，或扩展支持的安全环境变量引用 |
| Model | `CONTROL_PANEL_MODEL_ID` |

先确认扩展发送的是哪一种具体请求：

- Chat Completions 的最终路径通常是 `/v1/chat/completions`。
- Responses 的最终路径是 `/v1/responses`，官方 Codex 扩展属于这一类。
- 扩展若只支持固定服务或不允许自定义地址，则不能按本节接入。

不要把完整请求路径直接填入只要求 Base URL 的输入框。更多字段模板见
[手动配置](/docs/thq-api/configuration)，地址差异见
[端点说明](/docs/thq-api/endpoints)。

## Claude Code 扩展

Claude Code 的 VS Code 扩展与 CLI 共用 Claude Code 用户级配置。先按
[Claude Code 接入指南](/docs/thq-api/clients/claude-code)将 THQ 的 Base URL、Key 和模型
写入 `~/.claude/settings.json`（Windows 为 `%USERPROFILE%\.claude\settings.json`）。

随后在 VS Code 设置中搜索“Claude Code login”，开启 **Disable Login Prompt**。也可以打开
`Preferences: Open User Settings (JSON)`，在 VS Code 的用户设置中合并：

```json title="VS Code 用户设置"
{
  "claudeCode.disableLoginPrompt": true
}
```

若用户设置中已有其他字段，只合并 `claudeCode.disableLoginPrompt`，不要覆盖整个文件。
`ANTHROPIC_BASE_URL` 仍然写在 Claude Code 的用户级 `settings.json` 中，且必须使用不带 `/v1`
的主机地址。扩展会追加 Anthropic Messages 路径；手动添加 `/v1` 可能造成重复路径。

保存后运行 `Developer: Reload Window`，并新建 Claude Code 会话。`claudeCode.environmentVariables`
是扩展进程专用设置；THQ 这类需要与 CLI 共用的网关配置不要写在其中。详细说明见
[Claude Code 接入指南](/docs/thq-api/clients/claude-code)和
[Claude Code VS Code 官方文档](https://code.claude.com/docs/en/vs-code)。

## 保存并重新加载窗口

完成设置后：

1. 保存扩展设置或用户级配置文件。
2. 在命令面板运行 `Developer: Reload Window`。
3. 关闭旧的扩展会话，并创建新会话。
4. 如果修改了终端环境变量，完全退出 VS Code，再从读取到新变量的环境启动。
5. 检查扩展状态页、日志或配置诊断，确认 provider、Base URL 和模型已经更新。

只重新打开聊天面板不一定会重启扩展宿主，也不一定会重新读取环境变量。

## 验证

在新会话中发送一条最小提示词：

```text
请只回复：VS Code 已连接 THQ API
```

随后核对：

- 扩展显示的 provider 和模型是否符合预期。
- 最终请求路径是否与扩展协议一致。
- 控制台是否出现对应时间和模型的请求记录。
- 失败时是否有 HTTP 状态码、request ID 或已脱敏的错误信息。

## 常见问题

### 扩展设置中没有 Base URL

该扩展可能不支持自定义端点。查看扩展官方文档，不要假设隐藏字段、其他扩展的设置名或通用
`OPENAI_BASE_URL` 一定会生效。

### 官方 Codex 扩展配置后请求失败

确认修改的是 `~/.codex/config.toml`，provider 使用 Responses，并且 THQ 侧所选模型支持
`/v1/responses`。只支持 Chat Completions 的模型不能直接用于当前 Codex 自定义 provider。

### Claude Code 扩展仍使用旧地址

确认 `~/.claude/settings.json`（Windows 为 `%USERPROFILE%\.claude\settings.json`）的 `env`
对象包含 Base URL、Key 和模型，并在 VS Code 中开启 **Disable Login Prompt**。然后运行
`Developer: Reload Window`，关闭旧会话并新建会话。

### 返回 404

先确认扩展协议，再检查 Base URL 是否漏写或重复添加 `/v1`。OpenAI 兼容扩展使用 `/v1`
根地址；Claude Code 扩展使用不带版本后缀的地址。

更多排查步骤见[客户端接入总览](/docs/thq-api/clients/)和
[常见问题](/docs/thq-api/faq)。
